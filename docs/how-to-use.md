# How to Use This Book

This is not a document you read once and file away. It is an operating guide — something you return to when starting an Initiative, writing a Story, handling an incident, or planning a renewal.

## Who Should Read What

| You are... | Start here |
|------------|-----------|
| A **Product Manager** starting a new initiative | [Discovery Backbone](/upstream/discovery-backbone) → [Initiative Brief](/upstream/initiative-brief) |
| A **Developer** picking up a story | [Story Workflow](/downstream/story-workflow) → [Definition of Done](/downstream/definition-of-done) |
| A **QA Engineer** writing test scenarios | [Gherkin & AssertThat](/downstream/gherkin) |
| A **DevOps / SRE** handling an incident | [Incident Management](/onstream/incident-management) → [Post-Mortem Guide](/onstream/post-mortem) |
| A **Customer Success Manager** managing accounts | [Customer Lifecycle](/offstream/customer-lifecycle) → [Health Score](/offstream/health-score) |
| **Anyone** wanting the TL;DR | [10 Non-Negotiables](/reference/non-negotiables) |
| **New team member** understanding everything | Read linearly from [Framework Overview](/framework-overview) |

## Using the Cadences

Each phase has its own cadence table. The [Master Cadence](/reference/master-cadence) combines all ceremonies into a single reference. Bookmark it and refer to it when planning your quarter.

## Using the Templates

This book contains ready-to-copy templates for:
- [Initiative Brief](/upstream/initiative-brief)
- [Jira Epic, Story, Bug](/downstream/jira-templates)
- [Gherkin Scenarios](/downstream/gherkin)
- [Runbook](/onstream/runbook)

Copy the template. Fill in the blanks. Link everything.

## Using the Real Examples

The [Examples](/examples/rca-wallet-balance) section contains annotated real-world documents showing the framework applied to actual incidents and bugs. Read them to see the difference between a good artifact and a weak one.

## The Framework in One Sentence

> Build clarity before code. Operate with accountability. Close the feedback loop. Always.
